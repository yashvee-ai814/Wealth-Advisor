# Wealth Advisor — Backend

FastAPI backend powering the Wealth Advisor chatbot. Hosts a LangGraph agent with eight deterministic financial calculation tools and human-in-the-loop (HIL) tool approval.

---

## Folder structure

```
app/
├── main.py              FastAPI app, CORS, and middleware registration
├── session.py           WealthAdvisorState, MemorySaver checkpointer, make_config()
├── core/
│   ├── config.py        Environment settings (OLLAMA_BASE_URL, OLLAMA_MODEL)
│   ├── logger.py        Shared logger factory used across all modules
│   └── middleware.py    HTTP request/response logging middleware
├── agent/
│   ├── __init__.py      Exports the compiled graph
│   ├── graph.py         LangGraph StateGraph assembly with MemorySaver
│   ├── nodes.py         agent_node, human_approval_node, routing functions
│   └── tools.py         Eight financial calculation tools + ask_human
├── router/
│   ├── router.py        /health, POST /chat, DELETE /chat/{id}
│   ├── models.py        ChatRequest, ChatResponse, ToolCallInfo, PendingInterrupt
│   └── guardrails.py    Input/output safety checks
└── data/
    └── prompts.json     System prompt and guardrail response messages
```

## Prerequisites

- Python 3.12+
- [uv](https://github.com/astral-sh/uv) — `pip install uv`
- [Ollama](https://ollama.com) running locally with `gpt-oss:120b-cloud` pulled

## Local development

```bash
cd backend
uv sync
uv run uvicorn app.main:app --reload
```

API available at `http://localhost:8000`. Interactive docs at `/docs`.

## API reference

### `GET /health`

```json
{ "status": "ok", "model": "gpt-oss:120b-cloud" }
```

---

### `POST /chat`

Send a new message or resume after an interrupt.

**Request body:**

| Field | Type | Required | Description |
|---|---|---|---|
| `session_id` | string | yes | Unique session ID (generated by the frontend) |
| `message` | string \| null | one of these | User message for a new turn |
| `resume_input` | dict \| null | one of these | Payload to resume an interrupted graph |
| `auto_approve_tools` | bool | no (default false) | Skip HIL approval and run all tools automatically |

New message:
```json
{
  "session_id": "abc123",
  "message": "I'm 40, earn £70k, want to retire at 65 with £40k/yr"
}
```

Resume after tool-approval interrupt:
```json
{ "session_id": "abc123", "resume_input": { "approved": true } }
```

Resume after clarification interrupt:
```json
{ "session_id": "abc123", "resume_input": { "answer": "My pot is £30,000" } }
```

**Response:**

| Field | Type | Description |
|---|---|---|
| `session_id` | string | Echoed session ID |
| `reply` | string | Agent's text response |
| `status` | string | `complete`, `awaiting_tool_approval`, or `awaiting_clarification` |
| `pending_interrupt` | object \| null | Interrupt details when status is not `complete` |
| `tool_calls_used` | array | Tools that were called, with their inputs and outputs |

```json
{
  "session_id": "abc123",
  "reply": "Based on your inputs, your projected pot is £680,000...",
  "status": "complete",
  "pending_interrupt": null,
  "tool_calls_used": [
    {
      "name": "calculate_projected_pot",
      "args": { "current_pot": 30000, "monthly_personal": 500, "monthly_employer": 250, "annual_growth_rate": 0.05, "years": 25 },
      "result": "{\"projected_pot\": 681234.56, ...}"
    }
  ]
}
```

When `status` is `awaiting_tool_approval`, `pending_interrupt` looks like:
```json
{
  "type": "tool_approval",
  "tool_calls": [{ "name": "calculate_projected_pot", "args": {...} }]
}
```

When `status` is `awaiting_clarification`:
```json
{ "type": "clarification", "question": "What is your current pension pot value?" }
```

---

### `DELETE /chat/{session_id}`

Signals a session reset. The in-memory checkpointer has no delete API; the frontend generates a new `session_id` to start fresh.

```json
{ "status": "ok", "session_id": "abc123" }
```

## Financial tools

| Tool | Inputs | What it calculates |
|---|---|---|
| `calculate_projected_pot` | current_pot, monthly_personal, monthly_employer, annual_growth_rate, years | Future pension pot at retirement using `FV = PV*(1+r)^n + PMT*((1+r)^n-1)/r` |
| `calculate_drawdown_income` | pot_value, drawdown_rate, state_pension_annual | Annual retirement income: `pot * drawdown_rate + state_pension` |
| `calculate_monthly_savings_needed` | target_pot, current_pot, annual_growth_rate, years | Monthly savings to hit target: rearranged FV annuity for PMT |
| `calculate_shortfall` | income_goal, projected_annual_income | Annual income gap: `max(0, goal - projected)` |
| `calculate_readiness_score` | projected_income, income_goal | 0–100 score + label (On track ≥70, Needs attention 40–69, At risk <40) |
| `calculate_inflation_adjusted_goal` | current_goal, inflation_rate, years | Inflated goal in future money: `goal * (1+rate)^years` |
| `get_uk_state_pension_info` | current_age, retirement_age | £11,502/yr from age 67; returns £0 if retiring before 67 |
| `ask_human` | question | Pauses the graph and surfaces a clarification question to the user |

## Logging

All logs go to stdout in the format `YYYY-MM-DDTHH:MM:SS [LEVEL] name: message`.

| Logger | What it covers |
|---|---|
| `wealth_advisor` | App startup and shutdown |
| `wealth_advisor.http` | Every HTTP request: method, path, status, latency |
| `wealth_advisor.router` | Session IDs, guardrail hits, tool usage, response status |
| `wealth_advisor.agent` | LLM invocations, tool selections, approval decisions |
| `wealth_advisor.guardrails` | Blocked inputs and outputs with matched pattern details |

## Agent flow

```
User message
     │
  agent node ── LLM decides what to do
     │
     ├── ask_human called    → tools node (interrupt pauses graph, asks user for info)
     ├── other tools called  → human_approval node
     │                            ├── approved → tools node → back to agent node
     │                            └── rejected → back to agent node
     └── no tools            → END (final reply returned)
```
